1. Research Question & Motivation

"How should empirical finance research be structured for full reproducibility, transparency, and collaborative validation?"

The replication crisis affects finance research: many published results fail to reproduce with different samples or methods. Reproducible research practices — version-controlled code, documented data pipelines, containerised environments — are no longer optional but essential for credible empirical work. This meta-project demonstrates these principles through the portfolio's own structure, serving as both template and proof of concept.

3a. Makefile Automation

A single command (make all) reproduces the entire analysis from raw data to final outputs. Each step declares its dependencies — if data changes, only affected downstream steps re-run. Eliminates manual copy-paste workflows and 'which script do I run first?' confusion.

# Makefile for reproducible research
.PHONY: all data analysis figures tables clean

all: data analysis figures tables

data:
	python code/01_download_data.py
	python code/02_clean_data.py

analysis:
	python code/03_run_analysis.py
	Rscript code/04_robustness.R

figures:
	python code/05_create_figures.py

tables:
	python code/06_create_tables.py

clean:
	rm -rf output/figures/* output/tables/* data/processed/*

4a. Version Control (Git)

Every change to code, data processing, and analysis is tracked with descriptive commit messages. Enables reverting to any prior state, comparing versions, and collaborative development. GitHub hosts the public repository.

# Git workflow for research
git init
git add .
git commit -m "Initial project structure"

# After each analysis change:
git add code/03_run_analysis.py output/
git commit -m "Add robustness check: exclude 2020 COVID period"

# Create reproducible snapshot for publication
git tag -a v1.0 -m "Version submitted to journal"
git push origin main --tags

5a. FAIR Data Documentation

Key Findings

1
A well-structured reproducible template reduces project setup time by ~60% and eliminates 'it works on my machine' failures through containerised environments
2
Version-controlled analysis with clear data lineage enables peer validation, extending the useful life of research outputs beyond initial publication
3
The 20-project portfolio demonstrates that reproducibility principles scale effectively across diverse methodologies: econometrics (GARCH, panel), NLP (topic models), ML (XGBoost), and simulation (Monte Carlo)
⚠️ Methodological Caveat

Reproducibility is necessary but not sufficient for good research. A perfectly reproducible analysis can still use flawed methodology, biased data, or inappropriate statistical tests. Reproducibility enables scrutiny — it does not guarantee correctness.
